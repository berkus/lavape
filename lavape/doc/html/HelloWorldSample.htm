"Hello World" sample

Sample files

Please open HelloWorld.lava in LavaPE.

Topics

Initiator, exec, execution environment, basic types, String, MessageBox.

Overview

An initiator is a kind of "main program" or autonomous thread of execution in Lava. The usual "Hello World" sample is based on the initiator notion, the String class and its MessageBox member function.

Where to look and what to do

When you open a *.lava file in LavaPE (using the "File->Open" menu or by double-clicking it in Windows Explorer or by selecting it from the history list of the File menu) the Lava declaration tree view appears. The left splitter window of the declaration window lists all direct and indirect include files of the current file. You can minimize the include view by dragging its right border towards the left border of the declaration window.

In the declaration view the "HelloWorld" initiator is the first declaration of the (unnamed) top-level package: A Lava program file can be executed only if its first top-level declaration is an initiator.

In order to open the executable body of the HelloWorld initiator in the Lava exec view you may either double-click the little "exec" icon or select the Exec keyword to the right of the icon by a single mouse-click and then click the spectacles .

Try the "Tile windows" buttons in order to achieve a tiled or cascading arrangement of exec and tree views.

Click the "Run" button to run the HelloWorld.lava sample.

You should try to alter the initiator exec in order to familiarize with the Lava structure editor:

Change the "Hello World!" constant into a concatenation of two constants "Hello" and " World!":

  1. Click the "Hello World!" constant to select it (single mouse-click).
  2. Then click the "+" operator to make the constant the first operand of a "+" operation (= string concatenation, since the first operand is of type "String").
  3. Double-click the "Hello World!" constant, or select it and click the spectacles , or press the "down" arrow key to "open" it in a little text edit window.
  4. Now you can delete the trailing "World!" substring.
  5. Close the little edit window by pressing the return key or by clicking anywhere outside the window.
  6. Double-click the <expr> placeholder, or select it and click the spectacles, or press the "down" arrow key to open a little text edit window on it.
  7. Enter the string "World!".
  8. Close the little edit window and click "Run" again.

Assign "Hello " and "World!" to separate string variables:

  1. Select the call statement by clicking the call keyword and press the DEL button to delete the statement.
  2. Click the declare button on the keyword toolbar (to the left): The skeleton of a declare statement is inserted, with the selection on the <type> placeholder.
  3. Select type "String" from the "Basic types" combo-box. It replaces <type>, and the selection is on <varName> now.
  4. Double-click the <varName> placeholder, or select it and click the spectacles , or press the "down" arrow key to "open" it in a little text edit window. Enter the name of the first String variable (that shall hold the value "Hello "), e.g. "h".
  5. Press the return key to close the little edit window.
  6. Click the "Insert" button , or press the return key once more to insert another <var> placeholder for the second String variable. Enter its name, e.g. "w", in the same way as before. 
  7. Select the <stm> placeholder.
  8. Click the set keyword button to insert an assignment statement. The selection is on the <var> placeholder on the left-hand side of the assignment.
  9. Select the local variable "h" from the "Variables" combo-box. The selection is on <expr> thereafter.
  10. Double-click the <expr> placeholder, or select it and click the spectacles , or press the "down" arrow key to "open" it in a little text edit window. Enter the string "Hello ". Press the return key to close the little text edit window
  11. Click the set keyword of the assignment statement to select the latter. Repeat steps 8 - 10 to construct the second assignment statement "set w <== "World!".
  12. Click the set keyword of the second assignment statement to select the latter. Click the button to insert a function call statement after the second assignment statement.
  13. The selection is on the <expr> placeholder for the call object of the function call. The other placeholders of the function call statement are still black (= disabled).
  14. Click the "+" operator to replace <expr> with the envisaged string concatenation operation. The call object is now "(<expr> + <expr>)".
  15. Select the first operand of + and replace it with the variable "h" from the "(Variables)" combo-box.
  16. Select the second operand of + and replace it with the variable "w" from the "(Variables)" combo-box.
  17. The <func> placeholder is red (= enabled), since the type (String) of the call object can now be determined. Select it and replace it with function "MessageBox" from the "(Virtual call)" combo-box. Note that the latter contains only member functions of interface "String" (and of its base interface "Object").
  18. Test your changes by clicking the "Run" button .
