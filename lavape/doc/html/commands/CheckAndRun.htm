How to check and run Lava programs

A Lava file xy.lava that is to be executed must contain an initiator, whose declaration must be the first declaration in the (usually unnamed) top-level package of xy.lava.

Before you can start a Lava program you should make sure that it is

  • free of errors and
  • free of incomplete statements that still contain placeholders.

If you open a Lava file in LavaPE then a complete check of its contents is performed automatically, including all declarations and all execs and invariants. This same type of complete check can be carried out at any time during editing a Lava program by clicking the "Check" button . Use the "Check all" button to check also every directly or indirectly included file.

If a Lava program contains errors or placeholders then it can be started nevertheless. The Lava interpreter reads and checks Lava files only at the moment when they are really required for execution. The interpreter checks only the initiator to be started in a first step. If it then turns out at run time that another initiator is to be started or a new class implementation is to be instantiated then this single initiator or implementation is checked before it is used.

Click the "Run" button of LavaPE to save the current Lava file and to pass it to the Lava interpreter for execution.

You can launch a Lava program also in the usual way by double-clicking its name or icon in the Windows Explorer. You may open a *.lava program in LavaPE by selecting the menu item "Edit with LavaPE" from the Explorer's context menu on that *.lava file.

You may also double-click a *.lcom or *.ldoc file in Explorer to open the corresponding blank form or Lava document.

Finally it's also possible to launch LavaPE or the Lava interpreter from the command line (DOS window) by a command

Lava[PE].exe [name of a *.lava or *.lcom file] .
