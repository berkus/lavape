Sets and Chains

Lava provides two basic container classes "Set" and "Chain" (see the standard include file std.lava), with more or less obvious methods.

A Lava Set is actually a "multi-set": It may contain the same object more than once. ("Count" counts how often an object is contained in the set. For state objects it counts how many identical objects are in the set, for value objects how many set elements have the same value, i.e., the same contents.)

The type of the set elements is specified by the virtual type ELEMTYPE.

You can use the foreach, exists and select quantifiers to run through and process the elements of a set.

The Chain class is derived from Set and is essentially a doubly-linked list of elements. It gives you more control over the order of elements and the place of insertion/deletion.

Set and Chain elements can be identified through "handles" that may be returned on insertion.

In quantification constructs foreach, exists and select the handle operator @ may be used to get the handle of an object found by the construct. (The object found is represented by a local variable of the set element type.)

Sets and Chains, together with quantifiers foreach, exists and select, logical conjunctions and other constructs are the basis for the close integration of database functionality in Lava.
