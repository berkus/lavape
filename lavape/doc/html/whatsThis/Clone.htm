"Clone" expression

Use the clone expression to produce a complete copy of an existing Lava object.

Note that Lava provides a distinction between constituent and acquaintance members of an object. Only the constituents are copied by clone  ("deep copy"). Acquaintances are considered to be separate, independent objects. Clone copies only the pointers to acquaintances.

Like the new expression, clone provides a but clause where you can modify the cloned object before its creation state is switched to "finished". (Note that "finished" value objects are "frozen", i.e., they become constant and thus cannot be modified any more.)
