Member functions of Lava classes

Every function is either a static or a virtual member function of a Lava class. There are no global, "classless" functions in Lava.

If a member function is declared only in the implementation of the class then it can be referenced only from within this class implementation.

If it is declared in the class interface (= public part) of the class then it can be referenced from everywhere, except if it is declared "protected". In the latter case it can be referenced only from within the class implementation or the implementations of derived classes, and it can only be called with the "self" object.
