Release notes for release 0.9.2

I) Change Log for Release 0.9.2

A) Lava "virtual types" have been relieved from an unnecessary restriction

Before release 0.9.2 you had to observe the following rather restrictive rule:

If a class C is the value of a virtual type V then you can reference C only through this virtual type V within the containing pattern.

In release 0.9.2 we could completely get rid of this restriction: You may now reference C directly and may in particular create an object "new C" (which, however, will not be assignment-compatible to a variable of type V, but only to one of type C or of a base class ofC!). See also here.

But note: The other pertinent restriction, that a class C may be the value of only one virtual type, remains valid, since this unique VT serves as the type of the "self" variables of the methods of C.

 

B) Overriding of several virtual types in a derived class/package may now be performed successively

Before release 0.9.2 you had to perform this in a single step by selecting all virtual types that you wanted to specialize at once in the Lava override view, and then you had to click the override toolbutton (little yelow triangle pointing upward). Beginning from release 0.9.2 you may now override several virtual types successively in any order.

 

C) The Lava source code repository has been migrated from SVN to GIT.

Here the URL of the Lava GIT repository:

git://lavape.git.sourceforge.net/gitroot/lavape/lavape

 

II) Installation of LavaPE

For simplicity we have decided to provide binary releases of LavaPE only for Windows and Mac OS (Intel) in the future. There are so many Linux and other Unix variants that we shall provide only a single source tarball for all of these. Since we have migrated to the qmake build tool, the make process is so perfectly portable, that it should work without problems on most of these platforms, provided GNU make, GNU g++, and the developer version of the Qt4 framework have been installed before.

 

A) How to install the Windows binary release of LavaPE

Simply launch the downloaded lavape-<rel>-win32-bin.exe file and follow the instructions of the installer.

 

B) How to install the Mac OS (Intel) binary release of LavaPE

Download the most recent Qt4 release, presently

http://get.qt.nokia.com/qt/source/qt-mac-cocoa-opensource-4.6.2.dmg

and install it in the proposed default location.

Download the lavape-<rel>-macos.dmg disk image file and mount it by double-clicking. A double-click on LavaPE.app or Lava.app will launch the Lava Programming Environment or the Lava Interpreter, respectively.

For a more permanent installation, copy and paste the mounted image to your "Applications" folder or any other folder. Drag LavaPE.app/Lava.app to your dock, if you like.

A number of sample programs can be opened in LavaPE.app or Lava.app from the "samples" subdirectory of the installation directory "lavape".

C) How to build LavaPE from the sources on UNIX platforms

Download the compressed tar file lavape-<release>-src.tar.bz2 of the LavaPE source distribution and store it in the parent directory of the envisaged lavape installation directory. Uncompress it using the command

tar xjf lavape-<release>-src.tar.bz2

You need to have GNU g++/make and to get the developer version of Qt4 from Nokia (formerly TrollTech).

  1. On a Linux box, the software package manager of your Linux distribution usually offers the possibility to download and install the Qt4 development package(s).

    Slight disadvantage: You won't get the most recent Qt 4.6.x release this way, an the older 4.5.x releases unfortunately won't allow us to successfully invoke Qt Assistant, which is our online help tool, from LavaPE/Lava. This seems to work only for the Qt 4.6.x releases.

  2. Or you may download the binary version of Qt4 for Linux from http://qt.nokia.com/downloads/sdk-linux-x11-32bit-cpp and install it in the default directory /opt/qtsdk-<release>.

  3. Or you may download the Qt4 sources from the Nokia web site, currently at http://qt.nokia.com/downloads/linux-x11-cpp, and build it yourself.

In any case you must add the "bin" directory, where the Qt executables reside, to your PATH environment variable, if the Qt executables aren't located in a standard PATH directory anyhow.

In case 2 the actual Qt directory is the subdirectory  /opt/qtsdk-<release>/qt of the primary installation directory /opt/qtsdk-<release>. In case 3 the Qt directory is named Qt-<release> and is a sibling directory of the original source directory (i.e., it is located on the same level of nesting as the latter).

Then build the LavaPE executables from within the lavape installation directory as follows:

qmake lavape.pro
make

On some platforms the Qt4 version of qmake may have a different name (to distinguish it from the older Qt3 qmake).

On Mac OS X the qmake command should read

qmake -spec macx-g++ lavape.pro

On some Linux platforms the Qt4 version of qmake is named "qmake-qt4"

Execute "qmake -v" to make sure that you don't use the qt3 version accidentally (wIf you have installed the Qt development environment "QtCreator" then you can build LavaPE easily by opening the project file lavape.pro in QtCreator and issuing a "Build all" from the "Build" menu.

The build process will take quite a while. You will find the executables of the Lava Programming Environment in lavape/bin finally: "LavaPE", "Lava", and "LPC". (The Lava Precompiler "LPC" is needed only during the build process.) Go to the lavape/bin directory, launch "./LavaPE", consult the "what is", "what next" and online help, open some sample programs from the lavape/samples directory and enjoy the unique structure editing features of LavaPE.

 You can launch the Lava interpreter "Lava"

  • either by executing "./Lava" in the lavape/bin directory and by opening a *.lava or *.ldoc file from the file menu or by clicking the file open button,

  • or from LavaPE by clicking the "running man" toolbutton, after you have activated the executable code of a Lava main program or a declaration tree view that contains a main program declaration.

We'd appreciate very much if you would inform us about any problems that you have encounterd during the installation or while using Lava.

 

D) How to build LavaPE from the sources on Windows platforms

On Windows you may build LavaPE from the source distribution using VisualStudio VC++ [Express] 2008 or VisualStudio 2010 (which is available as a release candidate presently):

  1. Get the binary version of Qt4 from http://qt.nokia.com/downloads/windows-cpp-vs2008 and install it in a directory of your choice.

  2. Set the Windows environment variable QTDIR to point to the Qt4 installation directory. Add %QTDIR%\bin to your PATH environment variable (or else the Qt DLL's won't be found when launching LavaPE.exe/Lava.exe after the build process has finished).

  3. Get the LavaPE sources in the same way as in the UNIX case.

  4. Open the solution file <lavape-dir>\src\LavaPE.sln in VS2010 or <lavape-dir>\src\LavaPE-VS2008.sln in VS2008.ile <lavape-dir>\src\LavaPE.sln in VS2010 or <lavape-dir>\src\LavaPE-VS2008.sln in VS2008.

  5. Select the "Build Solution" command from the "Build" menu.

  6. After completion of the build process you can launch LavaPE.exe or Lava.exe from VC++: Just select the LavaPE or the Lava sub-project, respectively, as "StartUp Project" in the solution or class view first.

    Of course you may also launch LavaPE.exe or Lava.exe directly from the lavape\bin directory.
